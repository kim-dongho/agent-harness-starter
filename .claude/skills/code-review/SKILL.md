## /code-review 워크플로우

`/code-review` 로 호출한다. **3개 서브에이전트를 병렬로 띄워서** 각각 다른 관점으로 리뷰한 후 결과를 종합한다.

### 1. 변경 파일 확인

먼저 변경 파일 목록을 확인한다:

```bash
git diff HEAD --stat
```

변경이 없으면 "리뷰할 변경사항이 없습니다"를 출력하고 종료한다.

### 2. 서브에이전트 3개 병렬 실행

Agent 도구로 3개 서브에이전트를 **동시에** 띄운다. 각각 `git diff HEAD`를 읽고 자기 관점에서만 리뷰한다.

#### 서브에이전트 1: 보안 리뷰어
```
보안 전문가로서 `git diff HEAD`의 변경사항을 리뷰하라. 보안 관점만 집중:
- path traversal 가능성
- 미검증 입력
- 하드코딩된 시크릿
- 명령 주입 가능성
- 권한 문제
Critical/Important/Suggestion으로 분류하고, 파일:라인 번호를 포함하라.
```

#### 서브에이전트 2: 아키텍처 리뷰어
```
아키텍트로서 `git diff HEAD`의 변경사항을 리뷰하라. 설계/호환성 관점만 집중:
- OS 호환성 (macOS/Linux)
- 외부 의존성 문제
- glob 패턴 처리의 정확성
- 설정 기본값 일관성
- runner별 분기 처리
Critical/Important/Suggestion으로 분류하고, 파일:라인 번호를 포함하라.
```

#### 서브에이전트 3: 품질 리뷰어
```
시니어 개발자로서 `git diff HEAD`의 변경사항을 리뷰하라. 코드 품질 관점만 집중:
- 엣지 케이스 누락
- 중복 코드
- 에러 처리 누락
- 네이밍/가독성
- 테스트 커버리지
Critical/Important/Suggestion으로 분류하고, 파일:라인 번호를 포함하라.
```

### 3. 개별 리뷰 결과 출력

각 서브에이전트의 리뷰 결과를 **개별로** 먼저 출력한다:

```
---
## 🔒 보안 리뷰어

### Critical
- [파일:라인] 내용

### Important
- [파일:라인] 내용

### Suggestion
- [파일:라인] 내용

---
## 🏗️ 아키텍처 리뷰어

(동일 포맷)

---
## ✅ 품질 리뷰어

(동일 포맷)
```

### 4. 종합 리뷰 리포트

개별 결과를 종합한다:

- 중복 지적은 하나로 병합한다
- 2개 이상의 리뷰어가 동일 이슈를 지적하면 심각도를 한 단계 올린다
- 리뷰어 간 상충되는 의견이 있으면 명시한다

```
---
## 종합 리뷰 결과

### Critical (머지 차단)
| # | 이슈 | 지적 리뷰어 |
|---|------|:-:|
| C1 | [파일:라인] 내용 | 보안+아키텍처+품질 (3/3) |

### Important (논의 필요)
| # | 이슈 | 지적 리뷰어 |
|---|------|:-:|
| I1 | [파일:라인] 내용 | 보안+품질 (2/3) |

### Suggestion (비차단)
| # | 이슈 | 지적 리뷰어 |
|---|------|:-:|
| S1 | [파일:라인] 내용 | 아키텍처 (1/3) |

### 요약
- 변경 파일: N개
- 지적 사항: Critical N, Important N, Suggestion N
- 리뷰어 3/3 합의: N건 / 2/3 합의: N건 / 단독 지적: N건
```
