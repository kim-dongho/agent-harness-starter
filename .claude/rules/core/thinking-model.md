# Thinking Model

모든 작업 시작 시 복잡도를 판단하고, 그에 맞는 흐름을 따른다.

## Complexity

| 복잡도 | 기준 |
|--------|------|
| LOW | 파일 1~2개, 단순 수정 |
| MEDIUM | 파일 3~5개, 로직 변경 |
| HIGH | 파일 6개+ 또는 비즈니스 로직 변경 |

## Flow

- **LOW**: READ → IMPLEMENT → VERIFY
- **MEDIUM**: READ → ANALYZE → IMPLEMENT → VERIFY
- **HIGH**: READ → ANALYZE → PLAN(사용자 확인) → IMPLEMENT → VERIFY → REFLECT
