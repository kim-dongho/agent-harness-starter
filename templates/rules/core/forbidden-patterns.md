# Forbidden Patterns

아래 패턴은 코드베이스에서 금지한다. 발견 시 즉시 수정한다.

## 공통

- 시크릿/크리덴셜을 코드에 하드코딩하지 않는다 — 환경변수 또는 시크릿 매니저를 사용한다.
- `console.log`를 디버깅 용도로 남기지 않는다 — 프로덕션 코드에는 구조화된 로거를 사용한다.
- 주석 처리된 코드 블록을 커밋하지 않는다 — 삭제하고, 필요하면 git 히스토리에서 복원한다.
- `TODO`/`FIXME`는 이슈 번호 없이 남기지 않는다 — `TODO(#123): 설명` 형태로 작성한다.
- 요청 범위 밖의 코드를 "개선"하지 않는다 — 변경은 요청에 직접 연결되어야 한다.
- 기존 비즈니스 정책을 확인 없이 변경하지 않는다 — 정책 관련 코드는 반드시 확인 후 수정한다.
