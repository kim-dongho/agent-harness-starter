## 스마트 컨트랙트 보안 감사 체크리스트

### 접근 제어
- 모든 외부 함수에 적절한 접근 제어가 적용되어 있는가?
- 관리자 전용 함수가 보호되어 있는가?
- 권한 상승(privilege escalation) 경로가 없는가?

### 입력 검증
- 모든 외부 입력이 검증되는가? (zero address, 범위, 길이)
- overflow/underflow가 방지되는가?
- 재진입(reentrancy) 공격에 안전한가?

### 외부 호출
- 외부 호출 전에 상태가 업데이트되는가? (CEI 패턴)
- 외부 호출의 반환값을 확인하는가?
- delegatecall 대상이 신뢰할 수 있는가?

### 경제적 로직
- 토큰 decimal을 올바르게 처리하는가?
- 나눗셈에서 precision loss가 프로토콜에 불리하지 않은가?
- flash loan 공격에 안전한가?
- 오라클 조작에 대한 방어가 있는가?

### 가스 & DoS
- 무제한 루프가 없는가?
- 단일 실패가 전체 트랜잭션을 막지 않는가?
- 가스 비용이 예측 가능한가?

### 데이터 보안
- 온체인에 시크릿을 저장하지 않는가?
- 이벤트가 모든 상태 변경에 emit되는가?
- 서명 replay 공격에 안전한가?

### 배포 전 확인
- 정적 분석 도구를 실행했는가? (Slither, Mythril, Clippy)
- 테스트 커버리지가 충분한가?
- 프록시 패턴 사용 시 초기화가 안전한가?
- 메인넷/프로덕션 배포 전 외부 감사를 받았는가?
