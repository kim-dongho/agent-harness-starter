## 코드 리뷰 체크리스트

변경된 코드를 아래 순서로 검토한다. 각 항목에 대해 문제가 있으면 파일명과 라인을 포함하여 지적한다.

### 1. 정확성
- 로직 에러, off-by-one 실수
- 처리되지 않는 엣지 케이스 (null, empty, 경계값)
- 동시성/경쟁 조건
- 데이터 형태나 API 계약에 대한 잘못된 가정

### 2. 보안
- 미검증 사용자 입력 (SQL injection, XSS, command injection)
- 하드코딩된 시크릿/크레덴셜
- 누락된 인증/인가 체크
- 안전하지 않은 데이터 처리 (민감 데이터 로깅, 약한 암호화)

### 3. 성능
- 루프 내 불필요한 작업 (N+1 쿼리, 반복 계산)
- 캐싱 기회 누락
- 큰 메모리 할당 또는 메모리 누수
- async여야 하는 블로킹 연산

### 4. 에러 처리
- 무시된 에러 (catch 후 아무 처리 안 함)
- I/O 작업에 누락된 에러 처리
- 디버깅에 도움 안 되는 에러 메시지
- 실패 경로에서 누락된 리소스 정리

### 5. 유지보수성
- 하나의 함수가 너무 많은 일을 함
- 추출해야 할 중복 로직
- 불명확한 네이밍이나 맥락 부족
- 미사용 코드나 미사용 import

### 출력 형식

심각도별로 그룹화하여 제시한다:
- **Critical** — 머지 차단. 버그, 보안 취약점, 데이터 유실 위험
- **Important** — 논의 필요. 성능, 에러 처리, 로직 문제
- **Suggestion** — 비차단. 가독성, 네이밍, 스타일, 단순화 기회
