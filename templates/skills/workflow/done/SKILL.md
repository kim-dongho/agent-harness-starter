## /done 워크플로우

`/done` 으로 호출한다. 아래 품질 게이트를 순서대로 통과한 후 커밋 → MR을 생성한다.

### 품질 게이트 (5단계)

모든 게이트를 통과해야 다음 단계로 진행한다. 실패 시 원인을 수정한 후 재시도한다.

#### Gate 1. 코드 품질 (`/lint`)
```bash
{{LINT_COMMAND}}
```
lint, type-check가 통과하는지 확인한다.

#### Gate 2. 테스트 (`/test`)
```bash
{{TEST_COMMAND}}
```
변경 사항과 관련된 테스트를 실행한다. 실패 시 최대 3회 self-heal 루프.

#### Gate 3. 정책 보호
변경된 파일에 정책 키워드(날짜 계산, 금액, 상태 전이, 필터 조건)가 포함되어 있으면:
- 해당 정책을 검증하는 테스트가 존재하는지 확인한다
- 테스트가 없으면 작성을 요구한다

#### Gate 4. 범위 검증
```bash
git diff --stat
```
의도하지 않은 파일 변경이 없는지 확인한다.

#### Gate 5. 컨벤션
- 불필요한 파일(.env, .DS_Store 등)이 포함되지 않았는지 확인한다

### 커밋 (`/commit`)

변경사항을 프로젝트 커밋 컨벤션에 맞게 커밋한다.
