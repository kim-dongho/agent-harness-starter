## /analyze 워크플로우

`/analyze <기능명>` 으로 호출한다. 해당 기능의 도메인을 분석한다.

### 전제 조건

`docs/features/<기능명>/plan.json`이 존재해야 한다. 없으면 `/plan <기능명>`을 먼저 실행하라고 안내한다.

### 1. 도메인 용어집

프로젝트에서 사용하는 핵심 용어를 정의한다.

### 2. 기능 스펙 정의

plan.json의 각 태스크에 대해 상세 스펙을 작성한다:
- **Input/Output** — 입력과 출력 데이터 형식
- **예외 케이스** — 실패 시 동작
- **수락 기준** — 완료 조건

### 3. 산출물

```
docs/features/<기능명>/
├── plan.json      ← 이전 단계에서 생성
├── plan.md
├── glossary.json  ← 도메인 용어집
└── spec.md        ← 기능 스펙
```

### 4. 다음 단계

도메인 분석 후 → `/design <기능명>` 으로 설계를 진행한다.
