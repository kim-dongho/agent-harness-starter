## /analyze 워크플로우

`/analyze` 로 호출한다. 프로젝트의 도메인을 분석하여 용어와 기능 스펙을 정의한다.

### 전제 조건

`docs/plan.json`이 존재해야 한다. 없으면 `/plan`을 먼저 실행하라고 안내한다.

### 1. 도메인 용어집 생성

프로젝트에서 사용하는 핵심 용어를 정의한다. 에이전트가 동일한 용어를 일관되게 사용하도록 강제한다.

```json
{
  "domain": "프로젝트 도메인명",
  "terms": {
    "용어": {
      "definition": "정의",
      "aliases": ["다른 이름"],
      "context": "어디서 사용되는지"
    }
  }
}
```

### 2. 기능 스펙 정의

`docs/plan.json`의 각 기능에 대해 상세 스펙을 작성한다:

- **Intent** — 이 기능이 왜 필요한지
- **Input/Output** — 입력과 출력 데이터 형식
- **예외 케이스** — 실패 시 동작
- **수락 기준** — 완료 조건

### 3. 산출물

- `domain-glossary.json` — 도메인 용어집
- `docs/features/{기능명}.md` — 기능별 스펙 문서

### 4. 다음 단계

도메인 분석 후 → `/design` 으로 설계를 진행한다.
